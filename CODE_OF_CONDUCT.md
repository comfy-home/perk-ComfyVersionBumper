# Code of Conduct

Thank you for being part of the ComfyVersionBumper community.  
We aim to maintain a respectful, collaborative, and professional environment for all
contributors, users, and maintainers.

This Code of Conduct outlines expectations for participation and the process for
addressing unacceptable behaviour.

---

## 1. Our Standards

We expect all participants to:

- Be respectful and considerate  
- Communicate clearly and constructively  
- Assume good intent  
- Provide helpful feedback  
- Collaborate in a positive and solution‑focused manner  
- Respect differing viewpoints and experiences  
- Follow the project’s licensing and contribution rules  

We do **not** tolerate:

- Harassment, abuse, or personal attacks  
- Discriminatory language or behaviour  
- Threats or intimidation  
- Disruptive or hostile conduct  
- Spamming or low‑effort interactions  
- Attempts to bypass licensing restrictions  

---

## 2. Scope

This Code of Conduct applies to:

- GitHub Issues  
- Pull Requests  
- Discussions  
- Project communication channels  
- Any interaction representing the project or ComfyHome™  

It applies equally to contributors, maintainers, and users.

---

## 3. Reporting Unacceptable Behaviour

If you experience or witness behaviour that violates this Code of Conduct, please report it privately to:

**support@comfyhome.io**

Include:

- A description of the behaviour  
- Links or screenshots if available  
- Any relevant context  

Reports will be reviewed promptly and confidentially.

---

## 4. Enforcement

ComfyHome™ maintainers may take any action they deem appropriate, including:

- Warning the offender  
- Requesting corrective action  
- Temporarily or permanently banning participation  
- Closing or locking discussions  
- Rejecting or reverting contributions  

Repeated or severe violations may result in removal from the community.

---

## 5. Contributor Responsibilities

By participating in this project, you agree to:

- Follow this Code of Conduct  
- Respect the project’s licensing model  
- Avoid introducing harmful, insecure, or malicious code  
- Use GitHub Issues for problem reporting  
- Submit Pull Requests that follow the [CONTRIBUTING](https://github.com/comfy-home/perk-ComfyVersionBumper?tab=contributing-ov-file) guidelines  

---

## 6. Acknowledgement

By engaging with this project, you acknowledge that you have read and agree to abide by this Code of Conduct.

Thank you for helping keep the ComfyVersionBumper community respectful and productive.
